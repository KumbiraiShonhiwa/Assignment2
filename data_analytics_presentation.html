Data Sources & Structure

Three Primary Data Sources

1. Point of Sale (POS) Data

  • Transaction ID, timestamp, store location, product details, quantity, price
  • Customer ID (if loyalty member)

2. Online Orders Data

  • Order ID, timestamp, product details, delivery location, payment info
  • Customer ID (if logged in)

3. Loyalty Programme Data

  • Customer ID, join date, points balance, tier status
  • Purchase history, channel preferences

üìù Speaker Notes

Process: We began by examining the structure and schema of each data source. POS data provides in-store transaction details, online orders capture e-commerce activity, and loyalty data tracks customer engagement. The common key across these sources is Customer ID, which enables integration. Interpretation: Understanding data structure is crucial before any transformation. Notice that not all transactions have Customer IDs (non-loyalty customers), which requires careful handling during analysis to avoid biased comparisons.

Data Integration Strategy

Step 3: Creating Unified Dataset

pos_merged_with_members = pos_transactions.merge( loyalty_members, how="inner" ) online_merged = online_customers.merge( online_orders, how="inner" ) loyalty_merged = loyalty_members.merge( loyalty_points_activity, how="inner" ) pos_transactions_line_merged = pos_transactions.merge( pos_transactions_line, how="inner" ) line_products = pos_products.merge( pos_transactions_line, how="inner" )

üìù Speaker Notes

Process: This step focused on integrating related datasets using pandas.merge() with inner joins. Point-of-sale transactions were merged with loyalty member records, online customers were linked to their online orders, and loyalty members were connected to their points activity. Additional joins combined transaction headers with line items and product information to provide item-level detail.

Interpretation: Using inner joins ensures that only records with matching keys across datasets are retained, resulting in consistent and well-linked datasets. This integration creates a unified view of customer behaviour, transactional activity, and loyalty engagement, enabling accurate customer-level and product-level analysis.
